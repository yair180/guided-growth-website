Guided Growth OS
Five-region user flow. Memory is a shared resource. Analytics and Tools are cross-cutting.
How to read this view: A user interacts with the Frontend by speaking, tapping, or typing. Voice input flows through the Voice Layer for speech-to-text. Text and tap input skips Voice and goes straight to Backend. The Backend assembles a prompt with the user's context, calls the LLM, runs any tools the LLM requests, and sends the response back. The response flows back through Voice Layer (for spoken reply) or directly through Frontend (for text reply).

Memory is a shared resource. Frontend reads from it directly for auth and MP3s. Backend reads and writes it for everything else. Analytics records events from both Frontend and Backend. Tools are how the team builds and ships the system.

How the components combine: three paths

Different user moments use different combinations of the components above.
How to read this view: Same five regions as Zoom 1, opened up to show the actual vendors and components.

The Frontend is React + Vite hosted on Vercel, with three React hooks (useRealtimeVoice for Path 1, useAsync for Path 2, useLLM for Path 3). The Voice Layer is Vapi (orchestrator) plus Cartesia (Ink for STT, Sonic-3 for TTS, Sonic API for direct TTS). The Backend centers on callLLM() with the /api/context endpoint, tool execution, rate limiting and anonymization. The LLM is GPT-4o mini via OpenAI's API (swappable to Gemini, Groq, or Claude). Memory is Supabase: Auth (used by Frontend), Database (used by Backend), Storage (used by Frontend for MP3s).

The three paths below show how the components combine for different user moments: live voice for onboarding (Path 1), MP3 + LLM for daily check-ins (Path 2), text or async voice for free chat (Path 3).
