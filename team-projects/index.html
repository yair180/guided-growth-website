Live Project Boards Proposal / mock-up

A new operating model for the team. Each person stops hand-writing every line and instead runs several Claude sessions in parallel, one per project. Their real job is to define the requirements, let a Claude build it, and then verify the result — testing it, and going into the code when needed. It is the QA-engineer role: someone who knows code, directing AI instead of typing it.

Every project below is one Claude session, and its ordered steps are the running instructions that session works through. Drag steps to reorder, click a dot to change state, add a comment to any step — when you are signed in, changes save to the shared board (so the whole team and each Claude session see the same thing). Each bucket's Claude session reads its steps top to bottom and moves them as it goes.
