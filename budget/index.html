The model · three pools, not two

What changed on June 15

Until June 15 our subscription quietly subsidized agent runs at roughly 15 to 30 times under API price. That subsidy ended. Programmatic Claude now draws a separate metered credit. Here is where every kind of usage lands.

Pool 1 · unchanged

Interactive subscription

flat rate · $0 marginal
  • Claude.ai web, desktop, mobile
  • Claude Code in a terminal (interactive)
  • Board agents run as a foreground session
  • Workflows and subagents inside that session
June 15 does nothing here. This stays free against our flat limits.
Pool 2 · new

Agent credit

$100 / mo · metered at API rates · no rollover
  • Agent SDK and claude -p
  • GitHub Actions running Claude
  • Cron and background automation
  • Third party agents on our subscription
This is the $100. It bundles with one Max 5x plan and resets monthly.
Pool 3 · separate

Raw API key

pay as you go · own meter
  • OpenClaw on its own API key
  • Anything with its own ANTHROPIC_API_KEY
  • Never was subsidized
Unchanged by June 15. Tracked as its own line, not against the $100.

The four levers

How we stretch $100 across the whole month

1

Interactive over headless

Foreground terminal Claude stays on the flat plan. The moment we cron it as claude -p, every cycle starts billing. Worth more than the other three combined.

2

Per-person credit pools

Each teammate's own plan carries its own credit. Run agents under your own login and the team budget is the sum of everyone's credits, not one shared $100.

3

Haiku first for headless

Haiku is 5x cheaper than Opus per token. Reserve Opus for final synthesis only. This is a 5x dial on Pool 2 burn.

4

Cache and batch

Cache the board and context every cycle re-reads for 90% off input. Batch anything that can wait for another 50% off.

Why the levers matter · same job, different cost

One headless agent cycle, about 30k tokens in and 3k out. The two dials of model and cache move it across a 12x range. The same $100 buys anywhere from 440 to 5,500 cycles.
